Storm Chaser

Over the course of my life, there are many things that I have had to learn. When learning, there are always new things to try. And when compared to many other first tries, Storm Chaser holds up surprisingly well: the art, music, and programming was mostly done on my own and it does come together into a relatively polished final product.

Programming

When I was first looking back at the code I wrote for this project, I was happy to find early signs of myself following the single-responsibility principle.

However, upon closer inspection lots of poor code began to emerge - the inevitable decay of my once clean code over the course of a week. Hiding within the PlayerMovement class was a monolithic fixed-update loop handling sliding, jumping, movement, and collision.

Despite the several instances of poor code cleanliness, the code does what it needs to: make the player feel smooth to control. The variable-jump height in particular was very important to the game feel, considering it is one of your only options for avoiding death in the game. I think this also helped set it apart from the other 2D platformer entries in the game jam.

Music

One thing that I was particularly proud of about Storm Chaser is the dynamic music. I didn’t have enough time to write a very long musical track for the game, so instead I wrote a short one with several layers that would fade in and out. I imported these layers into a tool called FMOD and was able to fade them in according to the player’s progress in-game. I feel like it adds a lot to the progression of each level, and complements the build in difficulty as you progress through the level.

Difficulty

A quick note about the difficulty of this game - the “hard” mode is brutally hard. The fact that you can change the difficulty to “easy” makes that game playable, but still very difficult. Most of the difficulty comes at the end of the level: the obstacle spawner is programmed to speed up the further you get into the level, and the level design pushes you head-first into an upward climb against an oncoming wave of death. To make matters worse, there are no checkpoints.

If I could go back and change something about this game, it would probably be messing with the length and difficulty of its only level. A longer level through a more interesting enviroment with checkpoints would have made a more accessable and fun experience.

Conclusion

Overall, this jam taught me a lot of important lessons. The most important lesson was about project scope - I simply did not have enough time to produce all of the assets and create as many levels as initially planned. Some concessions need to be made if you want to achieve the final vision you have for the project. The other lesson that I took away from this jam is that game development is fun. I really enjoyed the time I spent working on this project despite (or maybe because of) the difficult workload, and was inspired to continue participating in jams.
